# CardFrame VM Cluster — AGENTS.md

## Quick start

```bash
cp .env.example .env     # edit HOST_IP, GRAFANA_CLIENT_SECRET, etc.
bash init.sh              # gen-auth → docker compose up → kc-setup → grafana-add-org --main → kc-add-user main admin <pass> grafanaAdmin admin@example.com → oauth-sync
```

Prerequisites on host: `curl`, `jq`, `docker`, **mikefarah/yq** v4.18+ (NOT the Python `yq`).

## .env

Required vars: `HOST_IP`, `KC_PORT`, `GRAFANA_PORT`, `KC_REALM`, `KC_BOOTSTRAP_ADMIN_PASS`, `KC_ADMIN_PASS`, `GRAFANA_CLIENT_SECRET`, `VMADMIN_PASS`.

`.env` is gitignored — always check `.env.example` for the canonical list.

## Architecture

vmauth is the single auth gateway (port 8427):

| Direction | Auth | Route |
|-----------|------|-------|
| **Read** | Basic Auth per-org | `vmauth → vmselect` — tenant from auth config (per-org credentials) |
| **Write** (internal) | Basic Auth `vmadmin` | `vmauth → vminsert` — tenant `0` (all data) |
| **Write** (tenant) | Basic Auth per-tenant | `vmauth → vminsert` — tenant from auth config |

Each tenant = Keycloak user in `{org}` group + vmauth auth entry + Grafana org. JWT `groups` claim maps to Grafana Org via `org_attribute_path=groups` and `org_mapping`. Role groups: `role-admin` → Grafana Admin, `role-editor` → Grafana Editor.

## Commands

| Action | Command |
|--------|---------|
| Setup admin org | `bash scripts/grafana-add-org.sh --main` |
| Add tenant org + group | `bash scripts/grafana-add-org.sh <org-name> <account_id> <display_name>` |
| Add tenant user | `bash scripts/kc-add-user.sh <org-name> <username> <password> <role> <email>` (role: admin/editor/viewer/grafanaAdmin) |
| Disable tenant user | `bash scripts/kc-delete-user.sh <username>` (re-enable in Keycloak) |
| Delete tenant user | `bash scripts/kc-delete-user.sh <username> --force` |
| Sync OAuth mappings | `bash scripts/grafana-oauth-sync.sh` |
| Regenerate auth config | `bash scripts/gen-auth.sh` |
| Hot-reload vmauth | `docker compose exec vmauth kill -HUP 1` |
| Import dashboards (skip existing) | `bash scripts/grafana-import-dashboards.sh <org-name> [--grafana-name <name>]` |
| Import dashboards (force overwrite) | `bash scripts/grafana-import-dashboards.sh <org-name> --grafana-name <name> --overwrite` |
| Import dashboards into main org | `bash scripts/grafana-import-dashboards.sh --main [--overwrite]` |

**Order matters** for new tenants: `grafana-add-org.sh` first (creates org + datasource + dashboards), then `kc-add-user.sh`.

**Org naming:** The first parameter `<org-name>` is the internal English name (used for vmauth auth, KC group name, dashboard UID suffix). The third parameter `<display_name>` is the Grafana org display name (can be Chinese). OAuth org mapping uses Grafana org ID (immutable), so renaming the org in Grafana UI won't break auth.

## File layout

| Path | Purpose |
|------|---------|
| `vmauth/auth.d/` | Per-tenant auth YAMLs (auto-generated by grafana-add-org.sh) |
| `vmauth/auth.yaml` | Merged output (gitignored, generated by gen-auth.sh) |
| `scripts/` | All management scripts |
| `docs/` | Architecture doc and Keycloak setup guide (Chinese) |
| `grafana/dashboards/default/` | Per-org dashboard templates (imported by grafana-add-org.sh) |
| `grafana/provisioning/datasources/` | (Not used — datasources created via API by grafana-add-org.sh) |
| `vmalert/rules/` | Alerting rules (health alerts for VM components) |
| `alertmanager/config/` | Currently routes to blackhole — configure receiver for real notifications |
| `vmagent/scrape.yaml` | Scrapes VM cluster component metrics |

## Gotchas

- Keycloak init is **two-phase**: bootstrap admin (disabled after setup) → permanent admin. Scripts are idempotent.
- vmauth `auth.yaml` is **not** checked into git — it is generated. Never edit it directly.
- Grafana datasource `vmauth-cluster` (uid `vmauth-cluster`) is created per-org by `grafana-add-org.sh` using Basic Auth (per-org credentials → vmauth → tenant-scoped Prometheus). All dashboard templates in `grafana/dashboards/default/` reference this UID.
- Admin user is created via the same flow as tenant users: `grafana-add-org.sh --main` then `kc-add-user.sh main admin <pass> admin <email>`. The `org-main` group maps to `Main Org.` in Grafana, and `role-admin` grants the Admin role.
- Alertmanager has no receiver configured (routes to `blackhole`). Configure a real receiver before expecting alerts.
